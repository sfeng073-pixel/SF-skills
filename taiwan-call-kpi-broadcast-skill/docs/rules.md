# 台湾通时通次播报规则

## 范围

当前专项只处理：

- 地区：`台湾`
- 小组：`台湾CC02组`
- 报表：`益智CC日通时通次监控`
- 时段：`15:00 / 17:00 / 21:30`

## 数据规则

- 统计区间：当天 `00:00:00` 到当前播报时段截止时间
- 15:00 截止到 `15:00:00`
- 17:00 截止到 `17:00:00`
- 21:30 截止到 `21:30:00`

## 排名规则

- 主排序：`通时` 降序
- 次排序：`通次` 降序
- 图片展示全员，不截断提醒逻辑

## 文字提醒规则

过程提醒统一按尾部管理：

1. 取 `通时` 最低 3 人
2. 取 `通次` 最低 3 人
3. 合并去重
4. 一位销售一行
5. 同时命中两类尾部时，行内写成 `通时尾部、通次尾部`

## 图片规则

- 图片只展示数据
- 不在图片里写提醒名单
- 台湾主题使用暖橙色系
- 表格字段固定：
  - 排名
  - 销售
  - 通时
  - 通时达标
  - 通次
  - 通次达标

## 钉钉发送顺序

1. 先发图片
2. 再发文字提醒

## 钉钉文字要求

- 支持 `@销售`
- 任意一位销售单独占一行
- 不输出“无异常也提醒一堆满足项”的噪音内容
- 如果当前没有尾部异常，可输出“当前无异常”

## 不做的事情

这个专项技能不负责：

- CRM 登录态维护
- 约课 KPI
- 出单与 GMV 播报
- 课前 2h / 课后 3h 跟进
- TL 日终总结
