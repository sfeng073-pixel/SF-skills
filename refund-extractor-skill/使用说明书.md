# 蓝凌OA退费数据提取工具 - 使用说明书

## 一、工具简介

本工具用于从蓝凌OA系统自动提取退费审批数据，导出为Excel文件，方便进行数据复盘、统计分析和报表生成。

## 二、适用场景

- 月度/季度退费数据汇总
- 退费原因分析
- 退费金额统计
- 学员退费情况追踪

## 三、系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS / Linux / Windows |
| Python版本 | 3.7 或更高 |
| 网络 | 可访问蓝凌OA系统 |
| 浏览器 | Chrome（用于获取Cookie） |

## 四、安装步骤

### 4.1 安装Python依赖

打开终端，执行：

```bash
pip install requests openpyxl
```

如果提示权限不足，使用：

```bash
pip install --user requests openpyxl
```

### 4.2 安装自动Cookie获取（可选但推荐）

为了让同事更方便使用，建议安装自动Cookie获取工具：

**方案1：browser_cookie3（最简单，从已登录浏览器读取）**

```bash
pip install browser_cookie3
```

安装后，只要同事在Chrome中登录了蓝凌OA，程序就能自动读取Cookie，无需手动复制。

**方案2：Playwright（自动打开浏览器窗口）**

```bash
pip install playwright
playwright install chromium
```

安装后，程序会自动弹出浏览器窗口，同事扫码/登录后按回车即可。

### 4.3 下载工具代码

将以下文件保存到本地目录，例如 `~/refund-extractor/`：

- `main_lanling_api.py`（主程序）
- `cookie_helper.py`（Cookie自动获取助手）

## 五、配置步骤

### 5.1 配置蓝凌OA域名

**方法一：修改代码（推荐）**

用文本编辑器打开 `main_lanling_api.py`，找到第31行：

```python
LANLING_BASE = os.environ.get("LANLING_BASE", "https://your-lanling-domain.com")
```

将 `https://your-lanling-domain.com` 替换为实际的蓝凌OA域名，例如：

```python
LANLING_BASE = os.environ.get("LANLING_BASE", "https://your-lanling-domain.com")
```

**方法二：设置环境变量**

在终端执行：

```bash
export LANLING_BASE="https://your-lanling-domain.com"
```

### 5.2 修改审批模板名称

找到第485行：

```python
if item.get('fdTemplate.fdName', '') != '【豌豆】退费申请':
```

将 `【豌豆】退费申请` 替换为实际的审批模板名称。

**如何查看模板名称？**

1. 登录蓝凌OA
2. 进入审批流程列表
3. 查看页面上的模板名称

常见模板名称示例：
- `【豌豆】退费申请`
- `退费审批流程`
- `学员退费申请`

## 六、首次运行

### 6.1 执行命令

```bash
cd ~/refund-extractor
python main_lanling_api.py --extract --days 7
```

### 6.2 获取Cookie（自动或手动）

程序会**自动尝试获取Cookie**，根据安装情况选择最佳方案：

#### 方案A：自动获取（推荐，如果安装了browser_cookie3）

如果同事已经在Chrome中登录了蓝凌OA，程序会自动读取浏览器Cookie，**无需任何操作**。

#### 方案B：自动获取（如果安装了Playwright）

程序会自动弹出浏览器窗口：

1. 等待浏览器窗口打开
2. 扫码或输入账号密码登录
3. 登录成功后，**按回车键**继续
4. 程序自动提取Cookie

#### 方案C：手动获取（如果自动获取失败）

**步骤1：打开蓝凌OA页面**

在Chrome浏览器中登录蓝凌OA（通常从钉钉工作台进入）。

**步骤2：打开开发者工具**

按 `F12` 或 `Cmd+Option+I` (Mac) 打开开发者工具。

**步骤3：获取Cookie**

切换到 **Console** 标签，输入：

```javascript
document.cookie
```

按回车，会输出一串字符串，例如：

```
"JSESSIONID=abc123; LtpaToken=xyz789; ..."
```

**步骤4：复制Cookie**

复制输出的完整字符串（包括引号内的所有内容）。

**步骤5：粘贴到终端**

回到终端，粘贴Cookie字符串，按回车。

### 6.3 验证登录

程序会自动验证Cookie是否有效：

```
[INFO] Cookie有效，已登录
```

如果显示登录失败，请检查：
- Cookie是否完整复制
- 蓝凌OA域名是否正确
- 网络连接是否正常

## 十、常见问题

### Q1: Cookie过期了怎么办？

**A:** 删除Cookie缓存文件，重新运行：

```bash
rm ~/.lanling_cookies.json
python main_lanling_api.py --extract --days 7
```

然后按提示重新输入Cookie。

### Q2: 提取的数据有重复？

**A:** 程序已内置去重机制，如果仍有重复：

1. 检查是否同时运行了多个实例
2. 删除Excel文件重新提取
3. 检查蓝凌OA是否有重复数据

### Q3: 某些字段为空？

**A:** 可能原因：

1. **历史数据**：2025年10月前的记录可能没有"豌豆学员ID"字段（表单升级前）
2. **未填写**：该字段在蓝凌OA中确实未填写
3. **解析失败**：HTML结构变化，需要更新代码

### Q4: 如何修改提取的字段？

**A:** 

1. 编辑 `main_lanling_api.py`
2. 找到 `RefundExcelHandler.DEFAULT_FIELDS` 列表
3. 添加或删除字段名
4. 同时更新 `extract_detail_from_html` 方法中的 `field_map` 映射

### Q5: 主管意见为什么提取不到？

**A:** 蓝凌OA的审批流程通过前端JavaScript动态渲染，不在服务端HTML中。需要通过其他方式获取，如：

- 钉钉审批API
- 浏览器自动化（Playwright/Selenium）

### Q6: 如何定时自动运行？

**A:** 使用系统的定时任务：

**macOS/Linux (crontab):**

```bash
# 每天上午10点运行
crontab -e
# 添加：
0 10 * * * cd ~/refund-extractor && python main_lanling_api.py --extract --days 1
```

**Windows (任务计划程序):**

1. 打开任务计划程序
2. 创建基本任务
3. 设置触发器（每天）
4. 设置操作（启动程序）
5. 程序路径：`python.exe`
6. 参数：`main_lanling_api.py --extract --days 1`
7. 起始于：`C:\Users\YourName\refund-extractor`

## 十一、故障排查

### 11.1 登录失败

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| Cookie无效 | Cookie过期或不完整 | 重新获取Cookie |
| 域名错误 | LANLING_BASE配置错误 | 检查并修改域名 |
| 网络超时 | 网络连接问题 | 检查网络，重试 |

### 11.2 获取列表为空

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 模板不匹配 | 模板名称错误 | 修改模板名称 |
| 无权限 | 账号无查看权限 | 联系管理员授权 |
| 时间范围错误 | days参数过大或过小 | 调整days参数 |

### 11.3 Excel保存失败

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 权限不足 | 无写入权限 | 检查目录权限 |
| 磁盘已满 | 磁盘空间不足 | 清理磁盘空间 |
| 文件被占用 | Excel文件已打开 | 关闭Excel后重试 |

## 十二、安全注意事项

1. **Cookie保密**
   - `.lanling_cookies.json` 包含登录凭证
   - 勿提交到Git仓库
   - 勿分享给他人

2. **数据脱敏**
   - 共享Excel前删除敏感信息
   - 如学员联系方式、家庭地址等

3. **访问控制**
   - 确保只有授权人员可以运行
   - 定期检查运行日志
