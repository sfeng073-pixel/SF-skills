# 营销海报生成器

> 完整的营销海报生成流程：市场调研 → 文案设计 → 视觉生成 → 迭代优化 → 交付

## 流程总览

```
用户输入主题 → 市场调研 → 调研报告 → 沟通细节 → 文案设计 → 生成海报 → 迭代优化 → 交付
```

## 第一步：市场调研（必须先执行）

### 触发条件
用户提出海报需求时，**必须先完成调研**，不得直接生成海报。

### 调研内容

使用 `WebSearch` 和 `WebFetch` 工具进行以下调研（控制在 3-5 次搜索以内）：

1. **同类海报设计趋势**
   - 搜索关键词模板：`{行业/领域} {主题/节日} 行銷海報 設計風格 {年份}`
   - 示例：`台灣 教育機構 618 行銷海報 設計風格 2025`

2. **目标市场本地化要点**
   - 搜索关键词模板：`{目标市场} {主题} 行銷文案 本地化`
   - 示例：`台灣 補習班 暑假 招生 文案 繁體中文`

3. **竞品参考**
   - 搜索关键词模板：`{行业} {主题} 海報 案例 優秀`
   - 示例：`教育科技 促銷海報 優秀案例`

### 调研报告输出格式

完成调研后，向用户输出结构化报告：

```markdown
## {主题} 营销海报调研报告

### 一、市场趋势
- 当前该主题/节日的营销热点
- 目标市场的文化习惯和注意事项
- 行业标杆案例参考

### 二、设计风格建议
- **色调**：推荐的配色方案及原因
- **风格**：适合的视觉风格（扁平/3D/手绘/极简等）
- **元素**：常用的视觉元素和符号

### 三、文案风格建议
- 目标市场的语言习惯（简繁体、用词差异）
- 常用营销话术和句式
- 禁忌用词和敏感内容

### 四、建议主题方向
| 主题 | 适用场景 | 文案示例 |
|------|----------|----------|
| ... | ... | ... |
```

## 第二步：沟通海报细节

调研报告输出后，使用 `AskUserQuestion` 工具向用户确认以下关键信息：

### 必须确认的信息

1. **产品/品牌信息**
   - 品牌名称
   - 是否有 Logo 或卡通形象（如有，请用户上传参考图）
   - 品牌主色调

2. **活动/主题细节**
   - 具体的营销主题（从调研建议中选择或自定义）
   - 优惠内容（具体折扣/课程/活动）
   - 目标受众（家长/学生/企业等）

3. **视觉偏好**
   - 喜欢的色调风格
   - 海报尺寸（默认 1080x1920 竖版）
   - 是否有参考图片

### 可选信息
- 投放渠道（FB/IG/LINE/实体印刷等）
- 是否需要多版本（A/B 测试）
- 文案特殊要求

## 第三步：文案设计

### 文案设计原则

1. **语言适配**
   - 台湾市场 → 繁體中文
   - 大陆市场 → 简体中文
   - 海外市场 → 英文或当地语言

2. **台湾繁体本地化要点**
   - 用词转换：优惠→優惠、活动→活動、体验→體驗、信息→資訊
   - 句式习惯：多用「讓」、「給孩子」、「一起」等亲切表达
   - 避免大陆用语：给力、牛逼、秒杀等

3. **文案结构**
   - 主标题：1 句话，吸引注意力
   - 副标题：1 句话，说明核心价值
   - 行动号召：明确的下一步引导
   - 品牌标语：强化品牌印象

### 文案输出
设计 2-3 套文案方案供用户选择，每套包含主标题、副标题、行动号召。

## 第四步：生成海报

### 使用 GenerateImage 工具

根据确认的方案调用 `GenerateImage` 生成海报图片。

### Prompt 编写要点

```
1. 明确说明品牌名称和文字内容
2. 详细描述卡通形象/Logo 的外观特征
3. 指定色调和视觉风格
4. 说明布局和构图要求
5. 强调目标市场的语言要求（繁体/简体）
6. 明确禁止出现的元素（如水印、多余文字）
```

### Prompt 模板

```
Marketing poster design for {品牌名} {活动主题} event, targeting {目标市场} market with {语言类型} text.

CHARACTER/MASCOT: {详细描述卡通形象外观}

BRAND: "{品牌名}" text logo at {位置}, {字体描述}, {颜色}

TEXT CONTENT (in {语言类型}):
- Main headline: "{主标题}"
- Sub-headline: "{副标题}"
- Badge: "{徽章文字}"
- Footer: "{底部标语}"

DESIGN STYLE:
- Background: {背景描述}
- Color palette: {配色方案}
- Decorative elements: {装饰元素}
- Mood: {整体氛围}

COMPOSITION:
- Brand logo at {位置}
- Main headline at {位置}
- Character at {位置}

IMPORTANT:
- All text in {语言类型}
- NO watermarks
- {其他禁止项}
```

## 第五步：迭代优化

### 生成后验证

使用 `Read` 工具查看生成的海报，检查以下项目：

| 检查项 | 标准 |
|--------|------|
| 品牌名称 | 正确显示，无多余图标/装饰 |
| 文字内容 | 无错别字、无多余文字、语言正确 |
| 卡通形象 | 清晰可辨，与参考图一致 |
| 整体风格 | 符合调研建议和用户偏好 |
| 水印 | 无 AI 生成水印 |
| 布局 | 层次清晰，重点突出 |

### 常见问题处理

| 问题 | 解决方案 |
|------|----------|
| 文字渲染错误 | 重新生成，在 prompt 中强调正确文字 |
| 多余图标/装饰 | 在 prompt 中明确禁止 |
| 色调不符合预期 | 在 prompt 中指定具体颜色 |
| AI 水印 | 在 prompt 中加入 "NO watermarks" |
| 卡通形象不准确 | 提供更详细的外观描述 |

### 迭代上限
- 最多迭代 **5 次**
- 如果 3 次仍无法解决同一问题，建议用户使用专业设计工具微调

## 第六步：交付

### 最终交付物

1. 海报图片文件（保存到 workspace 目录）
2. 海报评估报告（包含检查结果和改进建议）

### 文件命名规范
```
{品牌名}_{主题}_{版本号}.jpg
示例：VIPTHINK_618_v4.jpg
```

### 交付链接格式
```
[查看海报](computer:///sessions/.../workspace/{文件名}.jpg)
```

## 注意事项

1. **调研先行**：任何海报需求都必须先完成市场调研
2. **本地化优先**：根据目标市场自动适配语言和文化习惯
3. **用户确认**：关键决策点必须与用户确认后再执行
4. **质量把控**：每次生成后必须验证，不合格则迭代
5. **搜索预算**：调研阶段 WebSearch 不超过 5 次
